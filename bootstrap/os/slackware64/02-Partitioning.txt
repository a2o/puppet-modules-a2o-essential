###
### PARTITIONING AND MOUNTING
###



Destination disk needs partitioning.
You can either do it:
- manually, or
- use some of the tools/scripts to do in faster.

Then we need to mount this new filesystems (and few others) to their designated
locations.



###
### 1. Manual partitioning
###

No help here, except for command hints: fdisk, cfdisk, sfdisk, parted...

Once done, mount the following locations:
- /mnt/newroot      - future root and subordinate filesystems, like
- /mnt/newroot/var  - example subortinate fs
- /mnt/newroot/dev  - bind to /dev
- /mnt/newroot/proc - bind to /proc
- /mnt/newroot/sys  - bind to /sys
- /mnt/slack        - Slackware64 installation disk (/mnt/slack)

Now we are ready to run the automated installer.



###
### 2. Automated partitioning
###

In the scripts directory there is a script to automate the creation of
4Gswap-20Groot-RESTvar on /dev/sda, in no parameters are passed to it.

What is 4Gswap-20Groot-RESTvar? Here:
1. First create 4GB swap partition
2. Then create 20GB root partition
3. Then allocate all the remaining space to the /var partition

Like this:
--------------------------------------------------------------------------------
export A2O_BOOTSTRAP_URI="http://.../bootstrap/os/slackware64"

export A2O_BOOTSTRAP_FILE="autopartition.sh"
cd /root &&
rm -f $A2O_BOOTSTRAP_FILE &&
wget $A2O_BOOTSTRAP_URI/scripts/$A2O_BOOTSTRAP_FILE &&
chmod 755 $A2O_BOOTSTRAP_FILE &&
./$A2O_BOOTSTRAP_FILE

export A2O_BOOTSTRAP_FILE="automount.sh"
cd /root &&
rm -f $A2O_BOOTSTRAP_FILE &&
wget $A2O_BOOTSTRAP_URI/scripts/$A2O_BOOTSTRAP_FILE &&
chmod 755 $A2O_BOOTSTRAP_FILE &&
./$A2O_BOOTSTRAP_FILE
--------------------------------------------------------------------------------

If you want to use other drive than sda (sdb for example), do it like this:
./autopartition.sh /dev/sdb
